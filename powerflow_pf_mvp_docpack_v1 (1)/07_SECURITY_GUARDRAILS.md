# 07 — Guardrails и безопасность (MVP)

## 0) Цель

Не дать ассистенту:
- сканировать весь репозиторий,
- “ползти” в другие модули без явного решения,
- плодить оркестрационные скрипты.

---

## 1) Guardrail: module boundaries

### 1.1. Определение границ
Каждый module имеет:
- `root_path` (директория в repo)
- набор docs/plan/knowledge внутри `.pf/modules/<module_id>/`

### 1.2. Правило поведения ассистента
При работе по модулю:
- изменения допускаются **внутри root_path** (плюс, возможно, отдельные “shared” пути, если явно оговорены)
- если нужно изменение в другом модуле:
  - ассистент останавливается
  - объясняет “почему”
  - предлагает создать task в другом модуле
  - спрашивает подтверждение пользователя

pf не “запрещает” изменения на уровне git, но:
- pf context build выдаёт allowed_paths
- skills требуют не выходить за границы.

---

## 2) Guardrail: “не сканировать всё”

### 2.1. Единственный путь получения контекста
Assistent обязан использовать:
- `pf context build --intent ...`

Запрещено:
- “побегать по всему repo и прочитать всё подряд”
- хранить собственные индексы/скрипты извлечения (ad-hoc)

### 2.2. Ограничение поиска
Context Builder должен:
- использовать `allowed_paths` (из текущего module root или slice)
- ограничивать:
  - количество файлов
  - количество фрагментов на файл
  - общий budget (байты)

---

## 3) Guardrail: NO-SCRIPTS enforcement (социальный + технический)

### 3.1. Социальный
AGENTS.md + skills содержат явный запрет:
- “не добавляй python/скрипты для оркестрации”

### 3.2. Технический (минимум в MVP)
pf doctor должен уметь предупреждать:
- найдены новые файлы в `scripts/` или `tools/`, созданные в рамках активной задачи (если pf знает baseline)
- найдены новые CLI-обвязки, не зарегистрированные как pf artifact

> В MVP достаточно предупреждений + incident logging.

---

## 4) Secrets / PII

MVP правило:
- pf не логирует содержимое файлов в SQLite.
- Логи команд (artifacts) сохраняются как файлы; ассистент не должен вставлять в них секреты.
- При обнаружении секретов (например, токен в логе) ассистент обязан:
  - остановиться
  - предложить redaction
  - записать incident.

---

## 5) Операционная безопасность

- pf не требует network access.
- pf не запускает host.
- pf работает в пределах репозитория.
