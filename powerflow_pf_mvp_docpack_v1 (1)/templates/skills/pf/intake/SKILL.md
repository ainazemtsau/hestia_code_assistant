# Цель
Превратить “большую просьбу” пользователя в:
- mission (1 шт)
- 1–3 модуля для Milestone-1
- tasks (по модулю)
- focus на первый модуль

# Обязательные tool calls
1) `pf status --json`
2) `pf context build --intent status`

# Процесс

## 1) Сбор входных данных (вопросы)
Если запрос неполный — задай вопросы. Не более 5 за раз.
Формат:
- Вопрос
- Почему это важно (1 строка)
- Варианты ответа (если применимо)

## 2) Решение: patch vs mission
Если это маленький патч в одном месте:
- можно создать mission всё равно, но Milestone-1 = один модуль.

Если это “огромная задача”:
- создаём mission
- подробно планируем только Milestone-1
- Milestone-2+ только high-level

## 3) Определение модулей
Сначала попробуй получить кандидатов:
- `pf module detect --json`

Дальше:
- предложи пользователю 2–3 варианта разбиения (A/B/C)
- спроси подтверждение, какие модули берём на Milestone-1

Важно:
- НЕ пытайся “задокументировать весь проект” сразу.
- Если не уверен — спроси пользователя, какие части системы важны для задачи.

## 4) Создание данных в pf
1) Создай mission:
- `pf mission create --title "<коротко>"`

2) Для каждого выбранного модуля:
- `pf module upsert --module-id <id> --root-path <path> --display-name "<name>"`
- `pf module init --module-id <id> --write-scaffold`
- `pf task create --module-id <id> --title "<task title>"`

3) Установи focus на первый модуль:
- `pf focus module <id>`

## 5) NEXT
Скажи пользователю:
- **NEXT (Codex):** `$pf-planner`
- почему: “нужно составить план для первого модуля”

# Запреты
- Не пиши скрипты для роутинга/визардов.
- Не делай “авто-анализ архитектуры” всего repo.
