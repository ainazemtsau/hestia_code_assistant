# Обязательные tool calls
1) `pf status --json`
2) `pf context build --intent plan`

# Чеклист ревью

Проверь и верни список замечаний (если есть), иначе “OK”:

1) Ясность:
- можно ли понять план без тех. бэкграунда?

2) Варианты:
- есть ли 2–3 варианта?
- есть ли причины выбора?

3) Scope:
- scope ограничен модулем?
- non-scope явно указан?

4) Verification:
- есть ли команды проверки?
- они реалистичны (не “магия”)?

5) Slices:
- 1–5 шагов
- каждый шаг проверяемый
- нет “сделать всё”

Если есть замечания:
- предложи конкретную правку PLAN.md
- после правки: `pf plan mark-saved ...`
