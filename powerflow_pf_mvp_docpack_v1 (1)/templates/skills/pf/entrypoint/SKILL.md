# Контракт

- Всегда начинай с `pf status --json`.
- Дальше действуй строго по `next.kind/next.cmd`.
- Не создавай оркестрационные скрипты. pf уже даёт нужные команды для данных/контекста.

# Порядок работы

1) Выполни:
   - `pf status --json`

2) Сформулируй пользователю краткий отчёт (как менеджеру):
   - где мы сейчас (mission/module/task)
   - что блокирует (если есть)
   - что будет дальше

3) Если `next.kind == "skill"`:
   - Скажи пользователю: **NEXT (Codex):** `<next.cmd>`
   - И сам переключись на соответствующий skill (или попроси пользователя вызвать его).

4) Если `next.kind == "cli"`:
   - Скажи пользователю: **NEXT (CLI):** `<next.cmd>`
   - Поясни зачем (1 строка).

# Стиль общения
- Сухо, понятно, без “похвалы”.
- Если данных недостаточно — задавай вопросы.
